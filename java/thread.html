    # 1. Runnable

    Runnable规定了异步任务规范：无参、无返回值、无异常抛出。

    // 任何类如果想通过其他线程异步运行一个任务，就实现这个接口，实现run方法。
// 当然不一定非要实现接口，也可以通过传递函数参数形式实现
@FunctionalInterface
public interface Runnable {
    public abstract void run();
}

    # 2. Callable

    Runnable接口规定的异步任务没有返回值、没有异常，有时并不能满足需求。于是JDK又给出另外一个Callable规范。

    // Callable也定义了异步任务规范：可以有返回值，可以抛出异常
@FunctionalInterface
public interface Callable<V> {
    V call() throws Exception;
}

    # 3. Thread

    Runnable和Callable定义了异步任务方法签名，真正实现异步运行需要借助OS线程。在Java中如何使用线程，JDK提供了Thread这一类，其底层会调用OS函数实现线程的维护，Thread还实现了Runnable接口来。

    实现接口就需要实现接口定义的方法。Thread实现了Runnable就表明Thread也有任务需要异步运行，Thread也需要实现run方法，只不过Thread通过持有Runnable引用来定义自己的run方法。

    这种代码模式类似于代理方式，Thread类持有Runnable target对象并实现Runnable接口，而Thread类真正的run逻辑是target的逻辑。

    // 以下是Thread类的部分代码
public class Thread implements Runnable {
    private Runnable target;
    public Thread(Runnable target) {
        this(null, target, "Thread-" + nextThreadNum(), 0);
    }
    @Override
    public void run() {
        if (target != null) {
            target.run();
        }
    }
}

    Thread类的功能：调用系统函数创建线程，并异步运行target的run方法。

    这里有个问题：前面说通过Callable和Runnable都定义了异步任务规范，但是Thread类的构造方法却只支持Runnable实例。那么如何通过Callable接口创建线程?

    我们暂时无需关注Thread到底如何调用系统函数并创建线程，我们关注的是他是怎么实现执行我们异步任务的。 分析代码发现： （1）Thread类实现了Runnable接口，并持有一个Runnable实例的目标对象target成员变量 （2）Thread类实现run方法的逻辑，就是调用了target对象的run方法 （3）在创建线程时，可以将Runnable的实例当作参数传递给线程的构造方法，从而完成target属性的赋值 （4）也可以创建无target的线程,目的是干嘛？? 只要我们将需要执行的任务传递给创建的线程，线程就会被OS调用，就会在合适的时机调用我们任务的run方法。

    # 4. 通过Callable创建线程

    通过实现Runnable接口创建线程的方式比较简单，直接将Runnable接口实例传给Thread构造方法，线程就会异步执行我们的任务。

    之所以可以这么做的原因在于Thread实现了Runnable接口，且无需关注任务的结果（因为不返回结果）。

    而实现Callable接口却不一样，对于实现Callable接口的类，需要考虑如何异步运行任务，以及如何返回任务结果。

    # 4.1 Future接口

    Callable只定义了异步任务方法签名，对于任务结果，JDK又定义了一个接口Future，下面是Future接口定义的方法签名

    public interface Future<V> {
    // 取消任务
    boolean cancel(boolean mayInterruptIfRunning);
    // 判断任务是否取消
    boolean isCancelled();
    // 判断任务是否完成
    boolean isDone();
    // 获取任务结果
    V get() throws InterruptedException, ExecutionException;
    // 获取任务结果 含超时时间
    V get(long timeout, TimeUnit unit)
        throws InterruptedException, ExecutionException, TimeoutException;
}

    Callable是一个表示有返回值、有异常的异步任务接口；Future是一个表示异步任务结果的接口（现在只差一个实现类：异步运行Callable任务并返回Future对象，客户端拿着这个Future调用get()方法获取Callable任务结果）

    # 4.2 RunnableFuture接口

    Future是一个异步任务结果规范接口。实现Future的类除了需要实现上述异步任务的几个方法，还要考虑如何接收一个Callable任务，并执行该任务（你都不去调度任务怎么去获取结果）。

    接受一个Callable任务可以通过持有Callable引用实现。而异步执行任务JDK只提供了Thread一个类，Thread又只支持传递Runnable实例。

    没办法，实现Future的类还得实现Runnable，这样这个类才可以保证Callable的call方法被异步执行，并拿到任务结果。

    小结：实现Future的类型需要：（1）实现Future；（2）实现Runnable（3）持有Callable变量.在run()方法中调用call()方法，并完成任务结果的存储和Future接口方法的实现。

    为了更优雅地完成这件事，JDK又定义了一个RunnableFuture接口，这个接口就是将Future和Runnable缝合起来：同时实现Runnable和Future接口。

    实现Runnable需要编写能在Thread中异步运行的run()逻辑；实现Future需要实现任务结果的获取。

    于是乎，一切都很巧妙～

    /**
 * A {@link Future} that is {@link Runnable}. Successful execution of
 * the {@code run} method causes completion of the {@code Future}
 * Future表示run()方法成功执行的结果
 * 将Future和run()绑定在了一起
 */
public interface RunnableFuture<V> extends Runnable, Future<V> {
    /**
     * Sets this Future to the result of its computation
     * unless it has been cancelled.
     * 将这个Future当作这个异步任务计算的结果（除非任务被取消）
     */
    void run();
}

    这里可能会有一个问题：FutureTask接口为啥不去extends Callable而要去extends Runnable？

    回顾一下上面的知识点才能理出答案：

    （1）JDK只定义了一个Thread的类，用于和OS线程交互。这个类实现Runnable方法，持有一个Runnable target，实例化Thread的时候需要传递一个Runnable异步任务实例。

    换句话说只有满足Runnable接口的实例可以被Thread实例异步执行。所以Thread无法直接通过Callable实例创建线程并异步执行。

    （2）为了更好的规范异步任务结果，抽象了一个Future类，Future类可以获取任务结果、取消任务、判断任务是否结束

    （3）我们终究是为了：异步执行有返回值的任务，并返回任务的结果

    关键词：异步执行+任务结果。

    所以（1）RunnableFuture继承自Runnable方法，实现类实现run()方法，可以被线程异步执行；（2）继承自Future接口，实现类需要完成任务结果的处理；（3）至于为什么不继承自Callable，完全没必要，我们可以通过持有Callable引用来注入一个含有结果的异步任务。

    以下是FutureTask部分源码

    
// A cancellable asynchronous computation
// JDK将FutureTask称为可以取消的异步计算
// FutureTask可以开始、取消异步任务，可以获取异步任务结果
// 调用get()获取任务结果会阻塞当前线程，直到任务结束
// FutureTask异步任务一旦计算完成，无法再次start或者cancel，但是被runAndReset()调用除外
// FutureTask可以包装Runnable和Callable，因为FutureTask实现了Runnable且持有了Callable引用
// 详情可以参看他的构造方法
// FitureTask还提供了一些protected方法，供子类实现自定义任务类
public class FutureTask<V> implements RunnableFuture<V> {
    private volatile int state;// 任务状态以及中间状态
    private static final int NEW          = 0;
    private static final int COMPLETING   = 1;
    private static final int NORMAL       = 2;
    private static final int EXCEPTIONAL  = 3;
    private static final int CANCELLED    = 4;
    private static final int INTERRUPTING = 5;
    private static final int INTERRUPTED  = 6;
    // Callable引用
    private Callable<V> callable;
    // 任务结果
    private Object outcome;

    public FutureTask(Callable<V> callable) {
        if (callable == null)
            throw new NullPointerException();
        this.callable = callable;
        this.state = NEW;       // ensure visibility of callable
    }
    // FutureTask还可以接受Runnable实例
    public FutureTask(Runnable runnable, V result) {
        // Executors静态工厂方法，提供从Runnable实例到Callable实例静态方法
        this.callable = Executors.callable(runnable, result);
        this.state = NEW;
    }

    public void run() {
        // state为New
        if (state != NEW ||
            !RUNNER.compareAndSet(this, null, Thread.currentThread()))
            return;
        try {
            Callable<V> c = callable;
            if (c != null && state == NEW) {
                V result;
                boolean ran;
                try {
                    result = c.call();
                    ran = true;
                } catch (Throwable ex) {
                    result = null;
                    ran = false;
                    setException(ex);
                }
                if (ran)
                    set(result);
            }
        } finally {
            runner = null;
            int s = state;
            if (s >= INTERRUPTING)
                handlePossibleCancellationInterrupt(s);
        }
    }
}

    # 4.4 小结

    • Runnable代表无返回值的异步任务
    • Callable表示有返回值的异步任务
    • Future表示异步任务结果，可以是Runnable任务也可以是Callable任务
    • RunnableFuture表示异步计算，既有异步任务结果，同时也还是一个异步任务，可以提交给Thread或者线程池执行
    • FutureTask是RunnableFuture的实现类
    • FutureTask的源码核心：将FutureTask实例被Thread线程和当前线程共享，Thread线程运行run()逻辑，结束后设置state和outcome，get()方法阻塞当前线程直到state=NORMAL
    • 曾经为了记忆将FutureTask和Thread并列记忆，现在发现他们两个是两个概念。Thread是JDK提供的线程操作类，通过传递Runnable实例运行任务；FutureTask表示一个异步计算，可以提交给Thread运行，并通过get()方法获取任务结果。
