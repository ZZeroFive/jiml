    # 创建SpringBoot应用

    SpringBoot 3.x版本需要jdk 17及以上版本；spring boot 2.x版本需要jdk 11。

    spring 官方初始化程序无法再创建spring boot 2.x版本应用，如果想创建2.x 版本的Spring Boot项目，可以尝试以下几种方法：

    （1）升级JDK，使用SpringBoot 3.x,而后手动修改pom

    （2）创建普通maven项目，手动改造成spring boot项目

    （3）指定初始化中心，例如阿里云https://start.ailiyun.com

    个人采用第2种方式

    # 1.3 依赖介绍

    主要介绍一些SpringBoot应用研发过程会用到的一些依赖。 （1）dev tools spring boot dev tools是一个开发者工具模块，‌旨在提高开发效率和快速迭代开发。‌它提供了一系列功能，‌如自动应用程序重启、‌LiveReload支持、‌远程调试、‌启动热部署等。‌这些功能可以减少开发周期，‌增加开发体验，‌提高开发人员的工作效率。‌使用DevTools非常简单，‌只需将其添加为项目的依赖即可享受其中的各种功能

     <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-devtools</artifactId>
</dependency>

    (2) actutor Actuator是Spring Boot的一个子项目，‌它提供对应用程序的运维管理功能。‌通过访问和操作Actuator提供的端点，‌可以实现应用程序的监控和管理。

    ‌Actuator的好处在于提供了对应用程序内部运行状况的更深入控制和可视化，‌帮助开发人员更好地了解应用程序的状况，‌并及时进行优化和问题排查。‌

    Actuator通过RESTful API请求来监管、‌审计、‌收集应用的运行情况.例如可以访问:/actuator/health API获取服务健康状况，这个API常用于网关探测服务是否存活。

    # 1.4 指定打包方式

    maven打包方式不适合spring boot项目，spring boot有两种打包方式，一种是打fat包，包中含有依赖的信息；一种是打thin包，包中只有本服务的代码，依赖包则需要下载。

    <!--fat包，引用spring boot maven plugin即可-->
<plugin>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-maven-plugin</artifactId>
    <version>2.7.6</version>
</plugin>

<!--thin包就是指定一个依赖就可以了-->
<plugin>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-maven-plugin</artifactId>
    <version>2.7.6</version>
    <dependencies>
        <dependency>
        <groupId>org.springframework.boot.experimental</groupId>
        <artifactId>spring-boot-thin-layout</artifactId>
        <version>1.0.29.RELEASE</version>
        </dependency>
    </dependencies>
</plugin>
